  [33X[0;0YWhenever  the  object  constructor is called, it tries to simplify the given
  degree  list.  To  this  end  it  checks if subsequent degree group elements
  match. If so, their multiplicities are added. So, as in the example above we
  have:[133X
  
  
  [24X[33X[0;6Y[  [(1,0),2], [(1,0),3],[(0,1),2],[(1,0),1] ] \mapsto [ [ ( 1, 0 ), 5 ], [ (
  0, 1 ), 2 ], [ ( 1, 0 ), 1 ] ][133X
  
  [124X
  
  [33X[0;0YNote that, even though there are two occurances of [23X\left( 1,0 \right)[123X in the
  final  degree  list,  we  do not simplify further. The reason for this is as
  follows. Assume that we have a map of graded rows[133X
  
  
  [24X[33X[0;6Y\varphi \colon A \to B[133X
  
  [124X
  
  [33X[0;0Ygiven by a homomgeneous matrix [23XM[123X and that we want to compute the weak kernel
  embedding  of this mapping. To this end we first compute the row syzygies of
  [23XM[123X. Let us call the corresponding matrix [23XN[123X. Then we deduce the degree list of
  the  weak  kernel  object from [23XN[123X and from the graded row [23XA[123X. Once this degree
  list  is  known,  we  would  call  the  object  constructor.  If this object
  constructor  summarised  all  (and  not  only  subsequent) occurances of one
  degree  element in the degree list, then in order to make sure that the weak
  kernel embedding is a mapping of graded rows, the rows of the matrix [23XN[123X would
  have to be shuffled. The latter we do not wish to perform.[133X
  
  [33X[0;0YNote  that  the 'IsEqualForObjects' methods returns true whenever the degree
  lists  of two graded rows/columns are identical. So in particular it returns
  false,  if the degree lists are mere permutations of one another. Here is an
  example.[133X
